# Reports Archive

This folder stores implementation history, audits, verification notes, and phase-by-phase build logs.

It is intentionally separated from the main project documentation so the repository stays easier to read for portfolio and interview use.

## What belongs here

- phase implementation reports
- audit reports
- runtime verification logs
- cleanup reports
- observability stabilization notes

## What to read first

If you want the current project state, start with:

- [`README.md`](../../README.md)
- [`docs/guides/setup/GET_STARTED.md`](../guides/setup/GET_STARTED.md)
- [`docs/guides/setup/DEPLOYMENT_CHECKLIST.md`](../guides/setup/DEPLOYMENT_CHECKLIST.md)

## Archive structure

- `archive/portfolio-cleanup/`
  Portfolio cleanup and secret-hygiene reports

- `archive/phase-history/`
  Detailed phase-by-phase implementation and audit history

This archive is kept for traceability, but it is not the primary entry point for understanding the project.
